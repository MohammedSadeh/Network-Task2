HTTP بروتوكول

بروتوكول نقل النص التشعبي (HTTP)، وهو بروتوكول طبقة التطبيقات في الويب، هو جوهر الويب. وهو مُعرَّف في [RFC 1945] و[RFC 7230] و[RFC 7540]. يتم تنفيذ HTTP في برنامجين: برنامج عميل وبرنامج خادم. برنامج العميل وبرنامج الخادم، اللذان يتم تنفيذهما على أنظمة طرفية مختلفة، يتحدثان مع بعضهما البعض من خلال تبادل رسائل HTTP. يحدد HTTP بنية هذه الرسائل وكيف يتبادل العميل والخادم الرسائل. قبل شرح HTTP بالتفصيل، يجب أن نراجع بعض مصطلحات الويب. تتكون صفحة الويب (وتسمى أيضًا مستندًا) من كائنات. الكائن هو ببساطة ملف—مثل ملف HTML أو صورة JPEG أو ملف Javascrpt أو ملف جدول أنماط CCS أو مقطع فيديو—يمكن الوصول إليه من خلال عنوان URL واحد. تتكون معظم صفحات الويب من ملف HTML أساسي والعديد من الكائنات المرجعية. على سبيل المثال، إذا كانت صفحة الويب تحتوي على نص HTML وخمس صور JPEG، فإن صفحة الويب تحتوي على ستة كائنات: ملف HTML الأساسي بالإضافة إلى الصور الخمس. يشير ملف HTML الأساسي إلى الكائنات الأخرى في الصفحة بعناوين URL للكائنات. يحتوي كل عنوان URL على مكونين : اسم المضيف للخادم الذي يستضيف الكائن واسم مسار الكائن. على سبيل المثال، يحتوي عنوان URL http://www.someSchool.edu/someDepartment/picture.gif على www.someSchool.edu كاسم مضيف و/someDepartment/picture. gif كاسم مسار. نظرًا لأن متصفحات الويب (مثل Internet Explorer وChrome) تنفذ جانب العميل من HTTP، ففي سياق الويب، سنستخدم كلمتي المتصفح والعميل بالتبادل. تستضيف خوادم الويب، التي تنفذ جانب الخادم من HTTP، كائنات الويب، كل منها يمكن الوصول إليها من خلال عنوان URL. تشمل خوادم الويب الشائعة Apache وMicrosoft Internet Information Server. يحدد بروتوكول HTTP كيفية قيام عملاء الويب بطلب صفحات الويب من خوادم الويب وكيفية نقل الخوادم لصفحات الويب إلى العملاء. سنناقش التفاعل بين العميل والخادم بالتفصيل لاحقًا، ولكن الفكرة العامة موضحة في الشكل 2.6. عندما يطلب مستخدم صفحة ويب (على سبيل المثال، النقر فوق ارتباط تشعبي)، يرسل المتصفح رسائل طلب HTTP للأشياء الموجودة في الصفحة إلى الخادم. يتلقى الخادم الطلبات ويرد برسائل استجابة HTTP التي تحتوي على الأشياء.
